**Step 2: Create kernel build script**

`images/build-kernel.sh`:
```bash
#!/bin/bash
set -euo pipefail
KERNEL_VERSION="${1:-6.12}"
KERNEL_DIR="linux-${KERNEL_VERSION}"
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
OUTPUT_DIR="${SCRIPT_DIR}/../target/guest"

# Download kernel source if not present
if [ ! -d "$KERNEL_DIR" ]; then
    wget "https://cdn.kernel.org/pub/linux/kernel/v6.x/${KERNEL_DIR}.tar.xz"
    tar xf "${KERNEL_DIR}.tar.xz"
fi

cd "$KERNEL_DIR"
cp "${SCRIPT_DIR}/kernel/config-6.12" .config
make olddefconfig
make -j$(nproc) bzImage

mkdir -p "$OUTPUT_DIR"
cp arch/x86/boot/bzImage "$OUTPUT_DIR/vmlinux.bin"
echo "Kernel built: $OUTPUT_DIR/vmlinux.bin ($(du -h "$OUTPUT_DIR/vmlinux.bin" | cut -f1))"
```

**Step 3: Create rootfs init script**

**Step 4: Create rootfs build script**

```bash
#!/bin/bash
set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
OUTPUT_DIR="${SCRIPT_DIR}/../target/guest"
ROOTFS_DIR="${OUTPUT_DIR}/rootfs"

# Build static busybox if not present
# Or install from system package
mkdir -p "${ROOTFS_DIR}"/{bin,sbin,etc,proc,sys,dev,tmp,root,usr/bin,usr/sbin}

# Copy busybox and create symlinks
# (simplified: assumes busybox-static installed)
cp "$(which busybox)" "${ROOTFS_DIR}/bin/busybox"
chmod +x "${ROOTFS_DIR}/bin/busybox"
for cmd in sh ls cat cp mv rm mkdir mount umount ip; do
    ln -sf /bin/busybox "${ROOTFS_DIR}/bin/${cmd}"
done

# Copy init script
cp "${SCRIPT_DIR}/rootfs/init" "${ROOTFS_DIR}/init"
chmod +x "${ROOTFS_DIR}/init"

echo "Rootfs built: ${ROOTFS_DIR}"
```

### Task 3: Dynamic Resource Three-Piece Testing

**Output:**
- Create: `docs/resize-report.md`

This is the core validation of M0. All tests run against one long-lived VM across three dimensions.

**Step 1: CPU resize test**

```bash
# Start VM with CPU range
cloud-hypervisor \
    --api-socket /tmp/ch-api.sock \
    --kernel target/guest/vmlinux.bin \
    --cmdline "console=ttyS0" \
    --cpus boot=2,max=16 \
    --memory size=512M \
    --disk path=target/guest/rootfs.ext4

# In guest: run stress-ng to keep load
stress-ng --cpu 2 --timeout 0 &

# Resize loop (2 ↔ 16, 20 rounds each direction)
for i in $(seq 1 20); do
    # Scale up to 16
    time ch-remote --api-socket /tmp/ch-api.sock resize --cpus 16
    sleep 2
    # Scale down to 2
    time ch-remote --api-socket /tmp/ch-api.sock resize --cpus 2
    sleep 2
done
```

Record: each resize latency, failure count, guest CPU count verification (`nproc`).

**Step 2: Memory resize test**

Test three states:
- Idle: guest memory mostly free
- Under pressure: allocate 80% of current RAM with stress-ng
- Pinned: mlock a portion of memory

```bash
# Start VM with memory range
cloud-hypervisor \
    --api-socket /tmp/ch-api.sock \
    --memory size=512M,hotplug_method=virtio-mem,hotplug_size=32G \
    ...

# Resize loop (512M ↔ 8G, 10 rounds per state)
for state in idle pressure pinned; do
    echo "=== Memory resize: $state ==="
    for i in $(seq 1 10); do
        # Expand to 8G
        time ch-remote --api-socket /tmp/ch-api.sock resize --memory $((8*1024*1024*1024))
        sleep 1
        # Shrink to 512M
        time ch-remote --api-socket /tmp/ch-api.sock resize --memory $((512*1024*1024))
        sleep 1
    done
done
```

Record: expansion always succeeds; shrink success rate and latency per state.

**Step 3: Disk resize test**

Two paths:
1. `resize-disk`: expand existing virtio-blk disk, then `resize2fs` in guest
2. `add-disk`: hot-add a new disk

```bash
# 1. Resize existing disk
ch-remote --api-socket /tmp/ch-api.sock resize-disk --disk-id root --size 20G
# In guest: resize2fs /dev/vda

# 2. Add new disk
ch-remote --api-socket /tmp/ch-api.sock add-disk path=/tmp/extra.raw
# In guest: verify /dev/vdb appears
```

**Step 3: Write the client**

`crates/ch-client/src/client.rs`:
```rust
use std::os::unix::net::UnixStream;
use std::io::{Read, Write};
use crate::api::*;
use crate::error::{ClientError, Result};
use crate::error::ClientError::Api;

/// Client for Cloud Hypervisor's HTTP API over Unix domain socket.
pub struct ChClient {
    socket_path: String,
}

impl ChClient {
    pub fn new(socket_path: impl Into<String>) -> Self {
        Self { socket_path: socket_path.into() }
    }

    /// Send an HTTP request to the CH API socket.
    fn request(&self, method: &str, path: &str, body: Option<&str>) -> Result<String> {
        let mut stream = UnixStream::connect(&self.socket_path)?;
        let body_str = body.unwrap_or("");
        let req = format!(
            "{} {} HTTP/1.1\r\nHost: localhost\r\nContent-Type: application/json\r\nContent-Length: {}\r\n\r\n{}",
            method, path, body_str.len(), body_str
        );
        stream.write_all(req.as_bytes())?;

        let mut buf = Vec::new();
        stream.read_to_end(&mut buf)?;
        let response = String::from_utf8_lossy(&buf);

        // Parse HTTP response — extract body after \r\n\r\n
        if let Some(body_start) = response.find("\r\n\r\n") {
            let body = &response[body_start + 4..];
            Ok(body.to_string())
        } else {
            Err(Api("Invalid HTTP response".into()))
        }
    }

    /// Create a VM (POST /vm.create)
    pub fn vm_create(&self, config: &VmConfig) -> Result<VmInfo> {
        let body = serde_json::to_string(config)?;
        let resp = self.request("PUT", "/api/v1/vm.create", Some(&body))?;
        Ok(serde_json::from_str(&resp)?)
    }

    /// Boot the VM (PUT /api/v1/vm.boot)
    pub fn vm_boot(&self) -> Result<()> {
        self.request("PUT", "/api/v1/vm.boot", None)?;
        Ok(())
    }

    /// Shutdown the VM (PUT /api/v1/vm.shutdown)
    pub fn vm_shutdown(&self) -> Result<()> {
        self.request("PUT", "/api/v1/vm.shutdown", None)?;
        Ok(())
    }

    /// Resize vCPUs (PUT /api/v1/vm.resize)
    pub fn vm_resize(&self, desired_vcpus: Option<u8>, desired_ram: Option<u64>) -> Result<()> {
        let body = serde_json::json!({
            "desired_vcpus": desired_vcpus,
            "desired_ram": desired_ram,
        });
        self.request("PUT", "/api/v1/vm.resize", Some(&body.to_string()))?;
        Ok(())
    }

    /// Resize a disk (PUT /api/v1/vm.resize-disk)
    pub fn vm_resize_disk(&self, disk_id: &str, size: u64) -> Result<()> {
        let body = serde_json::json!({
            "id": disk_id,
            "size": size,
        });
        self.request("PUT", "/api/v1/vm.resize-disk", Some(&body.to_string()))?;
        Ok(())
    }

    /// Add a new disk (PUT /api/v1/vm.add-disk)
    pub fn vm_add_disk(&self, path: &str) -> Result<()> {
        let body = serde_json::json!({
            "path": path,
        });
        self.request("PUT", "/api/v1/vm.add-disk", Some(&body.to_string()))?;
        Ok(())
    }
}
```
